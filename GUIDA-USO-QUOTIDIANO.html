CRM Connecteed

Flusso operativo — dal Lead alla chiusura
v1.0 · Aprile 2026

1Lead

  • Sidebar → Leads → + Nuovo Lead
  • Compila: azienda, contatto, email, telefono, fonte
  • Assegna owner
  • Imposta Next Due Date (prossimo contatto)
  • Stato iniziale: New
  • Dopo primo contatto → Qualifying
  • Se interesse confermato → Qualified
  • Se non adatto → Unqualified (archiviato con nota)
▼

2Opportunità

  • Dal Lead Qualified → click Converti in Opportunità
  • Sistema crea automaticamente: Azienda, Contatto, Opportunità
  • Aggiungi prodotti/servizi dall'anagrafica
  • Imposta valore, probabilità %, data chiusura attesa
  • Gestisci la pipeline in Kanban (trascini tra le fasi)
  • Fasi: Discovery → Proposta → Negoziazione → Chiusura → Vinta/Persa
▼

3Preventivo

  • Dall'Opportunità → Crea Preventivo
  • Scegli il template (si caricano le voci in automatico)
  • Modifica quantità, prezzi, sconti
  • Aggiungi condizioni: tempi consegna, modalità pagamento, validità
  • Anteprima PDF per verificare
  • Click Invia al cliente → email + visibile nel Portale
  • Stati: Draft → Sent → Viewed → Accepted / Declined / Changes requested
  • Se cliente chiede modifiche → duplicare e creare nuova revisione (mai modificare quello inviato)
▼

4Contratto

  • Preventivo Accepted → sistema genera contratto bozza
  • Sidebar → Contratti → apri il contratto
  • Verifica clausole, date, importi
  • Imposta piano pagamenti: acconto (%) + milestone + saldo
  • Click Invia per firma (DocuSign)
  • Cliente firma online → stato Firmato
▼

5Fattura Acconto

  • Contratto firmato → tab Fatture → Genera Fattura Acconto
  • Sistema calcola IVA e genera numero progressivo
  • Verifica importo e dati
  • Invia al cliente (email + Portale)
▼

6Pagamento Acconto

  • Bonifico ricevuto → sidebar Fatture → apri fattura acconto
  • Click Registra pagamento
  • Inserisci: data, importo, metodo, riferimento bonifico
  • Stato fattura: Pagata
🔑 GATE ACCONTO — Automazioni attivate:
  • Il Progetto viene creato automaticamente (stato Ready)
  • PM e team ricevono notifica
  • Il cliente vede il progetto nel suo Portale
  • Prima di questo momento, nessun lavoro di delivery parte
▼

7Progetto (Delivery)

  • Sidebar → Progetti → apri il progetto creato
  • Tab WBS → verifica le fasi dal template, modifica se serve
  • Tab Gantt → definisci:
    • date pianificate di ogni task
    • dipendenze tra task
    • assegnazioni team
    • milestone
  • Click Imposta Baseline (fotografa pianificazione iniziale)
  • Stato progetto → In Progress
▼

8Time Tracking & Avanzamento

  • Team: sidebar → Time Tracking → + Nuova registrazione
    • Seleziona progetto, task, data, ore, descrizione
  • PM: nel Gantt aggiorna:
    • % completato di ogni task
    • stato: Todo / In Progress / Done / Blocked
  • % avanzamento progetto si aggiorna automaticamente
  • Task Blocked + motivo → notifica automatica a CEO
  • Task in ritardo → Proponi Riprogrammazione (ricalcola a valle)
▼

9Change Requests

  • Cliente chiede modifica fuori scope → MAI modificare direttamente il progetto
  • Apri il progetto → tab Change Requests → + Nuova CR
  • Titolo, descrizione, richiedente
  • Flusso stati:
    • Open → PM fa impact analysis
    • Impact Analysis → stima giorni + costo
    • Awaiting CEO → approvazione CEO
    • Awaiting Client → sistema genera preventivo addizionale + addendum
    • Approved → modifiche entrano nel progetto
    • Rejected → CR archiviata
▼

10Fatture Milestone

  • Milestone di pagamento raggiunta → sidebar Fatture
  • Genera da Contratto → seleziona la milestone
  • Verifica importo e invia al cliente
  • Pagamento ricevuto → Registra pagamento
  • Scheduler controlla fatture scadute ogni 15 min → stato Overdue + notifica automatica
▼

11Fattura Saldo

  • Tutti i task Gantt al 100%
  • Genera Fattura Saldo dal contratto
  • Invia al cliente
  • Registra pagamento finale
▼

12Chiusura Progetto

  • Saldo pagato → PM apre il progetto
  • Click Chiudi Progetto → stato Closed
  • Cliente riceve email + accesso al Portale con tutti i deliverable
  • (Opzionale) Attivazione contratto manutenzione/supporto ricorrente
▼

13Supporto Post-Delivery (Ticket)

  • Cliente apre ticket dal Portale
  • Sidebar → Ticket → nuovo ticket visibile con classe SLA + priorità
  • Click ticket → Assegna a te o collega
  • Rispondi (risposte inviate al cliente via email + Portale)
  • Stati: Open → In Progress → Waiting → Closed
  • Usa Canned Responses per risposte rapide ricorrenti
  • SLA controllato ogni 5 min → alert prima di sforare, escalation se sforato

★Portale Cliente

  • URL: https://crm.connecteed.com/portal
  • Il cliente vede:
    • Dashboard panoramica
    • Progetti con avanzamento %
    • Preventivi da accettare/rifiutare/modificare
    • Ticket (apertura e storico)
    • File marcati "Visibile al cliente"
  • Importante: file/ticket interni NON visibili al cliente finché non attivi il flag

★Report & Dashboard

  • Dashboard home: KPI in tempo reale (lead, pipeline, ticket, fatture)
  • Report disponibili:
    • Sales — vendite per periodo/commerciale
    • Pipeline — opportunità + valore pesato
    • Delivery — progetti, ritardi, carico team
    • Support — volume ticket, SLA compliance
    • Financial — fatturato, pagato, scaduto, previsionale
  • Tutti i report esportabili in Excel
  • Sidebar → Registro: audit log di ogni azione critica
